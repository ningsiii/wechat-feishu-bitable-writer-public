# 阶段 C 验收清单（单表上线版）

日期：2026-04-02  
目标：确认 V1-S（单表记录版）是否达到可演示/可售卖的最小标准。

---

## 0. 验收前提（必须满足）

- [ ] 网关已重启，且使用当前代码版本
- [ ] 当前只保留 1 张 active 表（单表模式）
- [ ] 飞书应用对该表有编辑权限
- [ ] 测试对话使用同一账号完成（避免跨会话干扰）

建议启动前环境（示例）：

```bash
export SMALLBIZ_FORCE_V2=1
export SMALLBIZ_ROUTER_MODE=v2
export SMALLBIZ_PROFILE=single_table
```

---

## 1. 核心功能验收（必须 100% 通过）

### 1.1 登记单表

- [ ] 发送 1 条飞书 base/wiki 链接，返回“已添加表”或“已登记”类回执
- [ ] 回执包含可点击链接
- [ ] 发送“列表”后，仅看到当前单表（或明确当前 active 表）

### 1.2 录入

- [ ] 发送一条订单文本，返回“已写入待确认（1条）”
- [ ] 回执包含：表格名 + 链接 + 确认/作废动作提示
- [ ] 飞书表中出现新行，确认状态为“待确认”

### 1.3 确认

- [ ] 发送“确认”，返回“已确认待确认（本次 N 条）”
- [ ] 飞书表中待确认记录状态变为“已确认”
- [ ] `ledger.jsonl` 记录 `pending_confirmed_all`

### 1.4 作废

- [ ] 在存在待确认记录时发送“作废”
- [ ] 返回“已作废最新一条待确认”
- [ ] 飞书表中对应待确认记录被删除

### 1.5 退出

- [ ] 发送“退出”返回“已退出录入模式”
- [ ] 退出后仍可正常执行“列表/登记/录入”

---

## 2. 异常场景验收（必须通过）

### 2.1 权限不足

- [ ] 对无权限表写入时，返回“执行失败 + 人话原因 + 修复建议”
- [ ] 不输出 Python Traceback 给最终用户

### 2.2 重复输入

- [ ] 60 秒内重复发送同内容，触发“可能重复”澄清
- [ ] 回复“继续”后只写入 1 次，不应出现连环澄清
- [ ] 回复“取消”后不写入

### 2.3 查询语句不应误写入

- [ ] 发送“查看所有水果记录/我都记录过哪些水果”类查询
- [ ] 不应新增待确认记录
- [ ] 不应在 sidecar 里新增该查询句 source_text

---

## 3. 输出体验验收（必须通过）

- [ ] 回复不出现“思考过程/让我检查一下/根据 AGENTS.md”等过程话术
- [ ] 表单助手路径下每条消息只返回一次最终回执（不连发多条）
- [ ] 回执格式统一：结果 / 表格 / 链接 / 动作提示（同风格）

---

## 4. 数据文件验收（必须通过）

- [ ] `draft.json` 状态与最近动作一致（active/session 不错乱）
- [ ] `ledger.jsonl` 事件与用户动作一一对应，无异常重复
- [ ] （若当前阶段禁用 sidecar）不应再写入 sidecar 新记录
